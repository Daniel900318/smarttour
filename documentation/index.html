2) Installation

Insert into HTML

You just add "bootstrap-md.min.css" and "bootstrap-md.min.js" to <head> and <body> accordingly in your bootstrap html file. Also add optional "bootstrap-md-toaster.min.js" if using toast message.


<head>
  <link rel="stylesheet" href="css/bootstrap.min.css">
  <link rel="stylesheet" href="css/bootstrap-md.min.css">
</head>
...
...
<body>
  <script src="js/bootstrap.min.js"></script>
  <script src="js/bootstrap-md.min.js"></script>
</body>

3) Basic Usage

BMD has its own class name with prefix bmd-* so that it is easy to add or remove material design stuff to or from element in your markup. For other usage of components, please view source markup on BMD demo page

4) Components Specification


MD color and utility classes

State color classes

Please note all color classes have !important to force overriding bootstrap rules.

Class name: bmd-{prop}-{state}
{prop} supports below CSS properties currently:
'text' - color
'bg' - background-color
'border' - border-color
'ink' - background-color (color of ripple)
{state} is 'default', 'primary', 'info', 'success', 'warning' or 'danger'

MD color classes

Class name: bmd-{prop}-{color}-[color-level]
{prop} supports below CSS properties currently:
'text' - color
'bg' - background-color
'border' - border-color
'ink' - background-color (color of ripple)
{color} is MD color listed as below table
[color-level] is MD color level and optional. It will use 500 if omitted.
For full list of color level, please refer to demo page or LESS file "material-design-color.less"

Floating class

Class name: bmd-floating
Just makes an element has shadow.

Example:


<p class="bmd-bg-green-50 bmd-text-green-900 bmd-border-success bmd-floating">
  <!-- This is an example to apply MD color classes -->
  <button class="btn btn-default bmd-ripple bmd-ink-pink">
    Ink in pink
  </button>
</p>

Dialogs - toast message

Usage

Only via JavaScript, the toaster plugin triggers a message box by appending <li> element (toast) to itself <ul> (toaster). The toast element <li> will be removed in document after being hidden.

A new toast must bounce at bottom right corner of screen with CSS3 animation and will automatically fade out after the time set in data-hide or manually via method.

Markup

You can predefine message, time to hide, bounce direction, style by simply specifying data-message data-hide data-bounce and data-styleclass correspondingly and please remember to add "bootstrap-md-toaster.js" as it is an independent plugin module. 


<ul class="bmd-toaster" data-message="Hello! I'm a toast message" data-hide="3"
    data-bounce="up" data-styleclass="success"></ul>
  ..
  ..
<script src="js/bootstrap-md-toaster.js"></script>

Method

$('#example').toaster(options)

Bounce a new toast message via JavaScript with options

.toaster('Hi! I'm different')

A shorthand to bounce a new toast with different message

.toaster('hide')

A shorthand to hide the lastest toast

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-bounce="up".

Name Type Default Description
message string ''

Default message content if data-message attribute isn't present.

It will be called with this reference set to the element that the toast is attached to.
bounce string 'up'

'up' : Bounce a new toast upwards.

'left' : Bounce a new toast from right to left.
hide number / string 4

Hide the toast after the seconds, or 'manual' to be hidden by method .toaster('hide'). It is based on the last in first out basis so as to always hide the latest toast at each call.
hidebutton boolean false

Show the hide button to allow user manually dismiss the message.
styleclass string 'default'

Set the background color of a new toast.

Same as state color scheme: 'default', 'success', 'info', 'warning', 'danger'. Even you can apply MD color classes such as bmd-bg-purple-100 and bmd-text-yellow-900

Or you may simply add your own color class as below example and then use in data-styleclass="my-color" or via JavaScript


.bmd-toaster > li.my-color {
  background-color: brown;
}

Events

Event Type Description
show.bmd.toast This event fires immediately when the .toaster() instance method is called.
shown.bmd.toast This event is fired when the toast has been made visible to the user (will wait for CSS transitions to complete).
hide.bmd.toast This event is fired immediately when the toast auto hide is triggered.
hidden.bmd.toast This event is fired when the toast has finished being hidden from the user (will wait for CSS transitions to complete). 

$('#myToaster').on('show.bmd.toast', function() {
  // do something...
})

Dialogs - Popovers

Because of different style, bootstrap default template is overrided by a new template. It is controlled by attribute data-bmd-state in the markup. You may still use bootstrap default by removing this attribute.

Javascript bootstrap-md.js will initialize all these state popovers for you. Please be reminded that $('[data-toggle="popover"]').popover(option); will initial again and override state template if execute in your own codes. To avoid this happen, you may use $('[data-toggle="popover"]:not([data-bmd-state])').popover(option); to initial bootstrap default popovers. 


$('[data-toggle="popover"][data-bmd-state]').popover({
  template: bmd_GLOBAL.popoverStateTemplate.replace('bmd-popover-state', 'bmd-popover-' + $(this).data('bmd-state'));
});

As you see the above initialization code, a global object bmd_GLOBAL has property popoverStateTemplate to store template as below and you can customize or replace it with your own design. The new template is a little bit different from bootstrap default. Just adding the state class bmd-popover-state for css style purpose.


<div class="popover bmd-floating bmd-popover-state" role="tooltip">
  <div class="arrow"></div>
  <h3 class="popover-title"></h3>
  <div class="popover-content"></div>
</div>

Dialogs - Tooltips

The approach is same as popover.

Javascript bootstrap-md.js will initialize all these state tooltips for you. Please be reminded that $('[data-toggle="tooltip"]').tooltip(option); will initial again and override state template if execute in your own codes. To avoid this happen, you may use $('[data-toggle="tooltip"]:not([data-bmd-state])').tooltip(option); to initial bootstrap default tooltips.


$('[data-toggle="tooltip"][data-bmd-state]').tooltip({
  template: bmd_GLOBAL.tooltipStateTemplate.replace('bmd-tooltip-state', 'bmd-tooltip-' + $(this).data('bmd-state'));
});

As you see the above initialization code, a global object bmd_GLOBAL has property tooltipStateTemplate to store template as below and you can customize or replace it with your own design. The new template is a little bit different from bootstrap default. Just adding the state class bmd-tooltip-state for css style purpose. 


<div class="tooltip bmd-tooltip-state" role="tooltip">
  <div class="tooltip-arrow"></div>
  <div class="tooltip-inner bmd-floating"></div>
</div>

Sidebar Nav

Sidebar navigation menu is responsive meta component that can be used together with bootstrap top navbar. It begins collapsed and toggleable in all viewports. To use it, html must has below basic markup and containing 3 component classes - bmd-sidebar, bmd-page-container, bmd-sidebar-toggle 


<div id="sidebar-example" class="bmd-sidebar bmd-sidebar-collapse bmd-sidebar-left">
  Your menu contents here... or use markup by copying from the below example
</div>
<div class="bmd-page-container">
  Your page contents here...
</div>
<a href="#" class="bmd-sidebar-toggle" data-target="#sidebar-example"></a>
Class name Description
bmd-sidebar-active The sidebar menu is shown.
bmd-sidebar-collapse, fix, overlay Define sidebar menu behavior. One of them must be specified in markup.
bmd-sidebar-small Small sidebar menu is still shown when collapsed. The default width is 62px defined in LESS variable @small-menu-width.
bmd-sidebar-left, right Position the sidebar menu. Either one must be specified in markup.
bmd-allow-overflow To use it, please set together with bmd-page-container. Allow page contents overflow with full width of viewport. Some page contents will be invisible when sidebar menu is active and you may make to scrollable with overflow attribute of body.
bmd-sidebar-xs, sm, md, lg Predefined width: xs=150px, sm=200px, md=250px (default), lg=350px. Their LESS variables are @sidebar-xs-width, @sidebar-sm-width, @sidebar-md-width, @sidebar-lg-width
bmd-sidebar-brand Different css style from menu items.
bmd-sidebar-default, primary, info, success, warning, danger State color for sidebar menu background. bmd-sidebar-default is used if not specified.
bmd-sidebar-toggle-default, primary, info, success, warning, danger State color for sidebar toggle. bmd-sidebar-toggle-default is used if not specified.

Tree View

Tree view is a component that can be used independently or together with sidebar nav. It begins with expanding first-level items and collapsible in all viewports. It is flexibe and easy to change style and icons. Two LESS variables @treeview-indent and @treeview-item-space to control branch indentation and each item spacing respectively. Three attributes data-expanded-icon, data-collapsed-icon and data-item-icon controls collapsible icon and item icon. To use it, html must contain 2 component classes - bmd-treeview, bmd-treeview-root. Morph FAB

Morph FAB can transform into panel, card or even other elements. Below example is a morph FAB to image. It expanded as an image on load and will revert to FAB when clicking on it.

Basic Usage

There are 3 main component classes must be set in markup:
bmd-morph : Target element that FAB to be transformed into
bmd-morph-close : Element on click to trigger becoming back to FAB
bmd-morph-fab : Morph FAB on click to transform into target element

Two initial states - expanded or closed can be set on load.
1. Expanded: add class bmd-morph-expanded to element bmd-morph
2. closed: add class bmd-morph-closed to element bmd-morph

Markup


<div class="row">
  <div class="col-lg-4">
    <div id="morph-img" class="bmd-morph bmd-morph-expanded">
      <!-- target element should be a 'block' display -->
      <a style="display:block" href="#" class="bmd-morph-close"><img class="img-responsive" src="images/work1.png"></a>
    </div>
  </div>
</div>
..
..
<span>
  <a href="#" class="btn btn-primary bmd-fab bmd-morph-fab bmd-ripple bmd-floating"
    style="position:fixed;bottom:90px;right:20px" data-target="#morph-img" data-sticktarget="true">
    <span class="bmd-icon flaticon-mark1"></span>
  </a>
</span>
..
..
<script src="js/bootstrap-md-morphFab.min.js"></script>

Method

$("#example").morphFab(options)

Create a new instance of morph FAB with options

.morphFab("expand")

Transform into target element

.morphFab("close")

Revert to FAB state

Options

Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-sticktarget="true".

Name Type Default Description
sticktarget boolean false Reserve column space for target element so that FAB always morph target at the same column. Otherwise, append to the end of row.
stickfab boolean false Only apply to FAB parent being <li>. Same as "sticktarget". Reserve original order for FAB so that FAB always home in the same order. Otherwise, append to the end of <ul>.
closemenu boolean false Only apply to FAB belonging to bmd-sidebar, bmd-bottom-sheet or bmd-fab-speed-dial-container. Menu will be auto closed when FAB left or homed.

Events

Event Type Description
expanded.bmd.morphFab This event is fired when target element is completely transformed (will wait for CSS transitions and animations to complete).
closed.bmd.morphFab This event is fired when FAB is completely reverted from target element (will wait for CSS transitions and animations to complete).

7) Customization

If you would like to change BMD, you are strongly recommended to amend LESS files instead of stylesheet because mixins are very useful to auto generate css rules. BMD contains four core LESS files:

  1. bootstrap.less - original bootstrap core file that is added import MD color palette
  2. bootstrap-md.less - BMD core styles and skin colors
  3. material-design-color.less - MD color palette
  4. variables.less - original bootstrap variables that is changed for state colors

Below steps show how to compile LESS files

  1. Copy all above LESS files from the folder "bmd-1.4.2/less" to the folder "bootstrap-3.3.5-less" with replace option.
  2. Compile "bootstrap.less" file in the folder "bootstrap-3.3.5-less" to generate "bootstrap.min.css" by using LESS compiler.
  3. Compile "bootstrap-md.less" file in the folder "bootstrap-3.3.5-less" to generate "bootstrap-md.min.css" by using LESS compiler.

For javascripts, BMD contains two core js files:

  1. bootstrap-md.js - core js to initial and handle BMD components such as ripple, sidebar nav
  2. bootstrap-md-toaster.js - only initial and handle toast message component
