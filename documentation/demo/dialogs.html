Popovers

Because of different style, bootstrap default template is overrided by a new template. It is controlled by attribute data-bmd-state in the markup. You may still use bootstrap default by removing this attribute.

Javascript bootstrap-md.js will initialize all these state popovers for you. Please be reminded that $('[data-toggle="popover"]').popover(option); will initial again and override state template if execute in your own codes. To avoid this happen, you may use $('[data-toggle="popover"]:not([data-bmd-state])').popover(option); to initial bootstrap default popovers. 


$('[data-toggle="popover"][data-bmd-state]').popover({
  template: bmd_GLOBAL.popoverStateTemplate.replace('bmd-popover-state', 'bmd-popover-' + $(this).data('bmd-state'));
});

As you see the above initialization code, a global object bmd_GLOBAL has property popoverStateTemplate to store template as below and you can customize or replace it with your own design. The new template is a little bit different from bootstrap default. Just adding the state class bmd-popover-state for css style purpose. 


<div class="popover bmd-floating bmd-popover-state" role="tooltip">






</div>

    Usage

    Only via JavaScript, the toaster plugin triggers a message box by appending <li> element (toast) to itself <ul> (toaster). The toast element <li> will be removed in document after being hidden.

    A new toast must bounce at bottom right corner of screen with CSS3 animation and will automatically fade out after the time set in data-hide or manually via method.

    Markup

    You can predefine message, time to hide, bounce direction, style by simply specifying data-message data-hide data-bounce and data-styleclass correspondingly and please remember to add "bootstrap-md-toaster.js" as it is an independent plugin module. 

    

      
..
..
<script src="js/bootstrap-md-toaster.js"></script>

      Method

      $('#example').toaster(options)

      Bounce a new toast message via JavaScript with options

      .toaster('Hi! I'm different')

      A shorthand to bounce a new toast with different message

      .toaster('hide')

      A shorthand to hide the lastest toast

      Options

      Options can be passed via data attributes or JavaScript. For data attributes, append the option name to data-, as in data-bounce="up".

      Name Type Default Description
      message string ''

      Default message content if data-message attribute isn't present.

      It will be called with this reference set to the element that the toast is attached to.
      bounce string 'up'

      'up' : Bounce a new toast upwards.

      'left' : Bounce a new toast from right to left.
      hide number / string 4

      Hide the toast after the seconds, or 'manual' to be hidden by method .toaster('hide'). It is based on the last in first out basis so as to always hide the latest toast at each call.
      hidebutton boolean false

      Show the hide button to allow user manually dismiss the message.
      styleclass string 'default'

      Set the background color of a new toast.

      Same as state color scheme: 'default', 'success', 'info', 'warning', 'danger'. Even you can apply MD color classes such as bmd-bg-purple-100 and bmd-text-yellow-900

      Or you may simply add your own color class as below example and then use in data-styleclass="my-color" or via JavaScript

      
.bmd-toaster > li.my-color {
  background-color: brown;
}

      Events

      Event Type Description
      show.bmd.toast This event fires immediately when the .toaster() instance method is called.
      shown.bmd.toast This event is fired when the toast has been made visible to the user (will wait for CSS transitions to complete).
      hide.bmd.toast This event is fired immediately when the toast auto hide is triggered.
      hidden.bmd.toast This event is fired when the toast has finished being hidden from the user (will wait for CSS transitions to complete). 
      
$('#myToaster').on('show.bmd.toast', function() {
  // do something...
})
