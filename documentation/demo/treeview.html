Tree View


Tree view is a component that can be used independently or together with sidebar nav. It begins with expanding first-level items and collapsible in all viewports. It is flexibe and easy to change style and icons. Two LESS variables @treeview-indent and @treeview-item-space to control branch indentation and each item spacing respectively. Three attributes data-expanded-icon, data-collapsed-icon and data-item-icon controls collapsible icon and item icon. To use it, html must contain 2 component classes - bmd-treeview, bmd-treeview-root. Whereas there are three optional classes - bmd-treeview-{state} controls stateful color, bmd-treeview-{style}-line controls line style, bmd-treeview-branch if any branches

Class name Description
bmd-treeview Mandatory component class. Set in element <ul>
bmd-treeview-root Mandatory component class. Set in element <li>
Three attributes data-collapsed-icon, data-expanded-icon and data-item-icon can be set in this element if need. They apply to all branches under this tree.
bmd-treeview-branch Optional component class. Set in element <li>
Three attributes data-collapsed-icon, data-expanded-icon and data-item-icon can be set in this element if need. They apply to all branches under this sub-branch.
bmd-treeview-{state} Optional class to control color. Where {state} is 'default', 'primary', 'info', 'success', 'warning' or 'danger'. default color is same as 'default' if this class is omitted.
bmd-treeview-{style}-line Optional class to control line style. Where {style} is 'solid', 'dotted', 'dashed' or 'hidden'. default style is 'solid' if this class is omitted.
